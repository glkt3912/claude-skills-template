# 目的

対象コードに対する包括的なテストを生成する。

# 手順

1. **対象コードの分析**
   - 関数/クラスの責務を理解
   - 入力/出力、副作用を特定

2. **テストケース設計**
   - `references/test-patterns.md` を参照
   - 正常系、異常系、境界値

3. **テストコード生成**
   - プロジェクトのテストパターンに従う
   - AAA パターンで記述

# 基本構造

```typescript
describe('関数名/クラス名', () => {
  it('正常系: 期待通りの動作', () => {
    // Arrange
    // Act
    // Assert
  });

  it('異常系: エラーハンドリング', () => {
    // ...
  });

  it('境界値: エッジケース', () => {
    // ...
  });
});
```

# 参照

- `references/test-patterns.md` - テストパターン集

# 制約

- 実装の詳細ではなく、振る舞いをテスト
- テスト同士は独立させる
- テストコードも可読性を重視
- 過度なモックは避ける
