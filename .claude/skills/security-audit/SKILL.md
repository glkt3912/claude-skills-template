# 目的

OWASP Top 10 を中心としたセキュリティ観点でコードを監査する。

# 手順

1. **対象の特定**
   - 監査対象のファイル・機能を確認
   - 技術スタック（フレームワーク、ライブラリ）を把握

2. **脆弱性チェック**
   - `references/owasp-checklist.md` に基づく検査
   - フレームワーク固有の脆弱性パターン確認

3. **レポート作成**
   - 発見した脆弱性を重要度別に分類
   - 修正方法を具体的に提示

# 出力フォーマット

```markdown
## セキュリティ監査レポート

### サマリー
- Critical: X件
- High: X件
- Medium: X件
- Low: X件

### 発見事項

#### [Critical] CWE-XXX: 脆弱性名
- **場所**: `ファイル:行番号`
- **問題**: 具体的な問題
- **影響**: 想定される影響
- **修正方法**: 具体的な修正コード
```

# 参照

詳細なチェックリストは `references/owasp-checklist.md` を参照。

# 制約

- 誤検知を減らすため、文脈を考慮して判断
- 修正方法は具体的なコード例を提示
- フレームワークの保護機能を考慮
