# 目的

Conventional Commits に従った英語のコミットメッセージを生成する。

# 手順

1. **変更内容の確認**
   - `git diff --staged` で差分を確認
   - 変更の目的・意図を把握

2. **コミットタイプの決定**
   - 変更内容に最も適したタイプを選択

3. **メッセージ作成**
   - 英語で簡潔に記述
   - 50文字以内を目安

# コミットタイプ

| タイプ | 用途 |
| ------ | ---- |
| `feat` | 新機能 |
| `fix` | バグ修正 |
| `docs` | ドキュメントのみ |
| `style` | フォーマット（動作に影響なし） |
| `refactor` | リファクタリング |
| `perf` | パフォーマンス改善 |
| `test` | テスト追加・修正 |
| `chore` | ビルド、CI、依存関係など |

# フォーマット

```
<type>(<scope>): <subject>

<body>
```

- **type**: 必須。上記タイプから選択
- **scope**: 任意。影響範囲（auth, api, ui など）
- **subject**: 必須。変更内容を簡潔に（英語、小文字開始、末尾ピリオドなし）
- **body**: 任意。詳細説明が必要な場合のみ

# 例

```
feat(auth): add password reset functionality

fix: resolve null pointer in user validation

docs: update API endpoint documentation

refactor(api): extract validation logic to separate module
```

# 制約

- 英語で記述
- 1行目は50文字以内を目安
- 命令形で記述（"Add" not "Added"）
- 「Generated by Claude」等の自動生成表記は絶対に入れない
- body が必要な場合は空行を挟む
