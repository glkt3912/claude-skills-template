# コードスメル一覧

## 肥大化

| スメル | 説明 | 対処 |
| ------ | ---- | ---- |
| 長いメソッド | 1つのメソッドが長すぎる（50行超） | メソッド抽出 |
| 大きなクラス | 責務が多すぎる | クラス分割 |
| 長いパラメータリスト | 引数が多すぎる（4個超） | オブジェクトにまとめる |
| データの群れ | 常に一緒に使われるデータ | クラスに抽出 |

## 変更の障害

| スメル | 説明 | 対処 |
| ------ | ---- | ---- |
| 変更の分散 | 1つの変更で複数ファイル修正 | 関連コードを集約 |
| 変更の発散 | 1つのクラスが複数理由で変更 | 責務を分離 |
| ショットガンサージェリー | 1つの変更が多くのクラスに影響 | 責務を統合 |

## 不要な複雑さ

| スメル | 説明 | 対処 |
| ------ | ---- | ---- |
| 怠惰なクラス | 存在価値が薄いクラス | インライン化 |
| 推測的な汎用性 | 使われない抽象化 | 削除 |
| 死んだコード | 使われないコード | 削除 |
| 複雑な条件分岐 | if/switch の入れ子が深い | ポリモーフィズム、早期リターン |

## カプセル化の欠如

| スメル | 説明 | 対処 |
| ------ | ---- | ---- |
| 特性の横恋慕 | 他クラスのデータに頻繁アクセス | メソッド移動 |
| 不適切な関係 | クラス間の過度な依存 | 依存関係の整理 |
| メッセージの連鎖 | a.b().c().d() のような呼び出し | 委譲メソッド |
| 中間者 | 処理を委譲するだけのクラス | 削除 |

## その他

| スメル | 説明 | 対処 |
| ------ | ---- | ---- |
| 重複コード | 同じコードが複数箇所 | 共通化 |
| マジックナンバー | 意味不明な数値リテラル | 定数化 |
| コメントの多用 | コードで表現できていない | リファクタリング |
