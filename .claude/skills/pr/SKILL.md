# 目的

日本語でわかりやすい Pull Request を作成する。

# 手順

1. **変更内容の把握**
   - `git log` でコミット履歴を確認
   - `git diff <base>...HEAD` で差分を確認
   - 変更の目的・背景を理解

2. **タイトル作成**
   - 日本語で簡潔に
   - 変更内容が一目でわかるように

3. **本文作成**
   - 概要、変更内容、動作確認方法を記載

# タイトルのフォーマット

```
[カテゴリ] 変更内容の要約
```

カテゴリ例:
- `[機能追加]`
- `[バグ修正]`
- `[リファクタリング]`
- `[ドキュメント]`
- `[テスト]`
- `[CI/CD]`

# 本文のフォーマット

```markdown
## 概要

この PR で何を実現するかを1〜2文で説明。

## 変更内容

- 変更点1
- 変更点2
- 変更点3

## 動作確認

- [ ] 確認項目1
- [ ] 確認項目2

## 関連 Issue

- #123
```

# 例

**タイトル:**
```
[機能追加] ユーザーのパスワードリセット機能を実装
```

**本文:**
```markdown
## 概要

ユーザーがパスワードを忘れた場合にリセットできる機能を追加します。

## 変更内容

- パスワードリセット用のAPIエンドポイントを追加
- メール送信サービスとの連携を実装
- リセットトークンの有効期限を24時間に設定

## 動作確認

- [ ] パスワードリセットメールが送信される
- [ ] トークン有効期限内にリセットできる
- [ ] 期限切れトークンでエラーになる

## 関連 Issue

- #45
```

# 制約

- タイトル・本文ともに日本語で記述
- 「Generated by Claude」「AI生成」等の自動生成表記は絶対に入れない
- 簡潔かつ必要十分な情報量
- レビュワーが理解しやすい構成
