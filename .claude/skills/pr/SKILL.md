# 目的

日本語でわかりやすい Pull Request を作成する。

# 手順

1. **変更内容の把握**
   - `git log` でコミット履歴を確認
   - `git diff` で差分を確認

2. **タイトル作成**
   - 日本語で簡潔に

3. **本文作成**
   - `templates/pr-template.md.tpl` を参照

# タイトルのフォーマット

```
[カテゴリ] 変更内容の要約
```

カテゴリ例:
- `[機能追加]`
- `[バグ修正]`
- `[リファクタリング]`
- `[ドキュメント]`

# 例

**タイトル:**
```
[機能追加] ユーザーのパスワードリセット機能を実装
```

# 参照

- `templates/pr-template.md.tpl` - PR 本文テンプレート

# 制約

- タイトル・本文ともに日本語で記述
- 「Generated by Claude」等の自動生成表記は絶対に入れない
- 簡潔かつ必要十分な情報量
