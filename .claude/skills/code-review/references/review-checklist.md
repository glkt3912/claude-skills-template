# コードレビューチェックリスト

## 正確性

- [ ] ロジックは正しいか
- [ ] エッジケースは考慮されているか
- [ ] null/undefined の処理は適切か
- [ ] エラーハンドリングは適切か
- [ ] 境界値の処理は正しいか

## 可読性

- [ ] 変数名・関数名は意図を表しているか
- [ ] 関数の長さは適切か（50行以下目安）
- [ ] ネストが深すぎないか（3段以下目安）
- [ ] コメントは必要最小限か
- [ ] 一貫したコーディングスタイルか

## 保守性

- [ ] 単一責任の原則に従っているか
- [ ] 重複コードはないか（DRY）
- [ ] 変更に強い設計か
- [ ] テストしやすい構造か
- [ ] 依存関係は適切か

## パフォーマンス

- [ ] 不要なループはないか
- [ ] N+1 クエリはないか
- [ ] メモリリークの可能性はないか
- [ ] 大量データの処理は適切か
- [ ] キャッシュは検討されているか

## セキュリティ

- [ ] 入力値は検証されているか
- [ ] 機密情報は適切に扱われているか
- [ ] SQL インジェクション対策はあるか
- [ ] XSS 対策はあるか
- [ ] 認証・認可は適切か

## テスト

- [ ] テストは追加/更新されているか
- [ ] エッジケースのテストはあるか
- [ ] テストは独立しているか
- [ ] モックは適切に使われているか

## ドキュメント

- [ ] 公開 API にドキュメントはあるか
- [ ] 複雑なロジックに説明はあるか
- [ ] 破壊的変更は記載されているか
