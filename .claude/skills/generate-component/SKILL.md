# 目的

プロジェクトの規約に従ったコンポーネントファイルとテストファイルを生成する。

# 手順

1. **既存パターンの確認**
   - `src/components/` 内の既存コンポーネントを確認
   - 命名規則、ディレクトリ構成、エクスポート方式を把握

2. **テンプレートの選択**
   - `templates/` 内から適切なテンプレートを選択
   - プロジェクトの技術スタックに合わせて調整

3. **ファイル生成**
   - コンポーネントファイル
   - テストファイル
   - 必要に応じて index.ts（バレルエクスポート）

4. **検証**
   - TypeScript エラーがないか確認
   - 既存のインポートパスと整合性があるか確認

# 制約

- 既存のコンポーネント構造に従う
- プロジェクトで使用しているテストフレームワークを使用
- 不要な依存関係を追加しない

# テンプレート

テンプレートファイルは `templates/` ディレクトリを参照:
- `react-component.tsx.tpl` - React コンポーネント
- `test.spec.ts.tpl` - テストファイル

# 使用例

```
ユーザー: UserProfileコンポーネントを作成して
実行: templates/ からテンプレートを読み込み、UserProfile用にカスタマイズして生成
```
